================================================================================
  SwapGo — AI 봇 매매 알고리즘 설명서
  (이 플랫폼을 만든 이유 + 어떤 봇이 어떤 알고리즘으로 매매하는가)
================================================================================

이 문서는 SwapGo 가 "무엇을 학습시키려고 만든 플랫폼인지" 와, 그 안에서 돌아가는
AI 봇들이 "정확히 어떤 매매 알고리즘으로 사고파는지" 를 코드 기준으로 설명합니다.
(구현체: swapgo-engine, 참고 코드 위치는 각 절 끝에 표기)

────────────────────────────────────────────────────────────────────────────────
[목차]
  0. SwapGo 를 만든 이유 (기획 의도)
  1. 큰 그림 — "과거 데이터로 학습한 모델"을 "모의시장에서 재현"한다
  2. AI 모델 4종 — 무엇을, 어떻게 학습했나
  3. 모델 입력 — 캔들에서 뽑는 8개 피처
  4. 모델 출력 → 매매 신호로 바꾸는 규칙
  5. 봇별 매매 알고리즘 (누가 어떻게 사고파나)
       5-1. BotA_Ingest   — AI 분석가 (신호/예측/심리 생성)
       5-2. BotA_Trade    — AI 자동매매 봇 (실제 매수/매도)
       5-3. BotB_Noise    — 시장 조성 봇 (거래량/유동성 공급)
       5-4. BotC_LP       — 유동성 공급 봇
       5-5. BotB_WS       — 실시간(WS) 분석 봇
  6. 매매 결정 파라미터 한눈에 보기
  7. 학습 → 배포 → 재현 파이프라인 (train/)
  8. 이 플랫폼으로 무엇을 배우는가

================================================================================
0. SwapGo 를 만든 이유 (기획 의도)
================================================================================
SwapGo 는 "모의투자 학습 플랫폼" 입니다. 진짜 돈을 쓰지 않고도

  (a) AMM(x*y=k) 기반 DEX 에서 스왑/유동성/슬리피지가 어떻게 작동하는지,
  (b) AI 트레이딩 봇이 과거 시장에서 학습한 패턴을 어떻게 실제 매매로 옮기는지,
  (c) 그 매매가 가격 차트·내 잔고·수익률에 어떻게 반영되는지

를 눈으로 보며 배우도록 만들었습니다.

핵심 컨셉은 **"AI 가 과거에 학습한 모델을 재현(reproduce)해서, 그 행동을 관찰하며
학습한다"** 입니다. 즉:

  · 진짜 Binance 의 과거 3년치 시세로 GRU 신경망을 미리 학습시켜 두고(오프라인),
  · 그 학습 결과(ONNX 모델)를 SwapGo 의 모의 시장에 띄워 "그대로 재현" 하고,
  · 봇이 내리는 매수/매도 판단과 그 결과(가격 움직임, 손익)를 사용자가 관찰합니다.

그래서 "어떤 봇이 어떤 알고리즘으로 매매했는가" 를 설명하는 것이 이 플랫폼의
교육적 핵심이며, 그 내용을 이 문서에 정리했습니다.


================================================================================
1. 큰 그림 — "학습한 모델"을 "모의시장에서 재현"한다
================================================================================

   [오프라인 · 과거]                         [온라인 · 실시간 재현]
   ┌───────────────────────┐                 ┌──────────────────────────────┐
   │ Binance 과거 5m 캔들   │   학습          │  swapgo-engine (:9000)        │
   │ (BTC·ETH 3년치)        │ ─────────────►  │   AIEngine 가 ONNX 모델 추론  │
   │   ↓ GRU 신경망 학습    │   .onnx 배포    │     ↓                         │
   │ model_*.onnx + scaler  │                 │   봇들이 그 예측으로 매매     │
   └───────────────────────┘                 │     ↓ /swap/execute           │
            train/                            └──────────────┬───────────────┘
                                                             ▼
                                            SwapGo 백엔드(:8000) AMM 풀 체결
                                                             ▼
                                            프론트 차트·보유자산·수익률에 반영

요점: 봇은 즉석에서 학습하지 않습니다. **이미 학습이 끝난 모델을 불러와(재현)**
실시간 캔들에 대해 추론만 합니다. 학습 자체는 train/ 의 오프라인 파이프라인에서
미리 끝내고, 그 산출물(.onnx, scaler.pkl)을 models/ 에 두면 봇이 재현합니다.

  · 학습 코드 : swapgo-engine/train/ (00~03 단계)
  · 추론 엔진 : swapgo-engine/ai/ai_engine.py
  · 모델 파일 : swapgo-engine/models/*.onnx, scaler.pkl


================================================================================
2. AI 모델 4종 — 무엇을, 어떻게 학습했나
================================================================================
모두 동일한 구조(GRU 순환신경망)로, Binance 5m 캔들로 학습한 뒤 ONNX 로 내보낸
모델입니다. 입력 길이(seq_len)와 "몇 봉 뒤를 맞추도록 학습했는가(타깃 horizon)"만
다릅니다.

  모델            seq_len   학습 타깃(미래)     역할
  ───────────────────────────────────────────────────────────────────────
  model_trade      30       5분 후 (1봉)        실제 자동매매 (BotA_Trade)
  model_scalper    10       1시간 후 (12봉)     단기 신호  → horizon "1h"
  model_swing      60       24시간 후 (288봉)   중기 신호  → horizon "24h"
  model_longterm   120      7일 후 (2016봉)     장기 신호  → horizon "7d"

  · 공통 입출력: 입력 (1, seq_len, 8) → 출력 (1, 2) = [BTC 로그수익률, ETH 로그수익률]
  · 출력은 EMA(alpha=0.3)로 평활해 노이즈를 줄임 (ai_engine.py).
  · GRU 하이퍼파라미터(학습 가이드 기준): hidden=64, layers=2, dropout=0.2,
    batch=512, lr=1e-3, early stopping. 방향 정확도 목표 BTC≥55% / ETH≥53%.
  · 모델 파일이 없으면 AIEngine 은 "Mock 모드"(직전 수익률을 그대로 반환)로 동작.

  · 참고: train/traning_guide.md, config.py(seq_len_*·model_*_path)


================================================================================
3. 모델 입력 — 캔들에서 뽑는 8개 피처
================================================================================
FeatureBuilder 가 BTC·ETH 캔들에서 매 봉마다 아래 8개 숫자를 계산하고, 학습 때
저장한 scaler.pkl 로 정규화해 모델에 넣습니다. (학습 순서와 정확히 동일해야 함)

  idx  피처              계산식 / 의미
  ───────────────────────────────────────────────────────────────────────
   0   Ret_btc           log(close_t / close_t-1)          BTC 로그수익률
   1   Ret_eth           log(eth_close_t / eth_close_t-1)  ETH 로그수익률
   2   Log_Pow_btc       log(volume_base + 1)              체결강도 근사(거래량 로그)
   3   Log_Trades_btc    log(trades_count + 1)             체결 건수
   4   Volat_btc         std(최근 10봉 Ret_btc)            단기 변동성
   5   BB_Width          4σ / SMA (20봉 볼린저밴드 너비)   추세/횡보 폭
   6   Volume_btc        volume_base (raw)                 BTC 거래량
   7   Volume_eth        eth_volume_base (raw)             ETH 거래량

  · scaler.pkl 이 없으면 정규화를 생략(Mock)하고 raw 값으로 추론.
  · 참고: ai/feature_builder.py


================================================================================
4. 모델 출력 → 매매 신호로 바꾸는 규칙
================================================================================
모델이 내놓는 건 "예측 로그수익률(ret)" 한 개(코인별)입니다. 이걸 사람이 이해하는
매수/매도/관망 신호와 신뢰도로 바꾸는 순수 함수들이 있습니다. (bot_a_ingest.py)

  [방향 판정]  _ret_to_signal(ret)
       |ret| < 0.0001(≈0.01%) → "hold"(관망)
       ret > 0               → "buy"  (매수)
       ret < 0               → "sell" (매도)

  [신뢰도]     _ret_to_confidence(ret) = 0.5 + 0.5·tanh(|ret| / 0.001)
       → 0.5(불확실) ~ 1.0(강한 신호). 예측이 클수록 confidence 가 1 에 가까워짐.

  [시장심리]   _ret_to_sentiment(ret) = tanh(ret/0.001)·100  →  -100 ~ +100 정수

  [보조지표]   같은 캔들로 RSI(14) · MACD(12,26,9) · MA7 · MA25 도 계산해 함께 제공.

  [앙상블]     BotA_Ingest 는 3개 모델(scalper/swing/longterm)의 ret 평균으로 최종
               방향·신뢰도를 산출. horizon 별 신뢰도 감쇠(1h=1.0, 24h=0.75, 7d=0.50)와
               예측구간(±0.5%/1.5%/3.0%)을 적용 — 먼 미래일수록 덜 확신.


================================================================================
5. 봇별 매매 알고리즘 (누가 어떻게 사고파나)
================================================================================
봇은 main.py 의 lifespan 에서 태스크로 동시에 기동됩니다(.env 토글로 on/off).
체결은 모두 봇 API 키(scope bot:trade/bot:lp)로 백엔드 /swap·/liquidity 를 호출.

─────────────────────────────────────────────────────────────────────────────
5-1. BotA_Ingest — "AI 분석가" (신호/예측/심리 생성, 기본 ON)
─────────────────────────────────────────────────────────────────────────────
역할 : 매매가 아니라 "분석 결과를 만들어 백엔드에 올리는" 봇. 프론트 /ai 페이지가
       이걸 보여줌. (즉 사람 트레이더에게 보여줄 AI 의 견해를 생산)
주기 : INGEST_INTERVAL_SEC (기본 30~60초)
알고리즘:
   1) BTC·ETH 5m 캔들 수집 → FeatureBuilder 로 8피처 행렬 생성
   2) 3개 모델(scalper/swing/longterm)에 적재 후 병렬 추론
   3) 코인별로 앙상블 → side(buy/sell/hold) + confidence + sentiment + RSI/MACD/MA
   4) horizon 1h/24h/7d 예측가격을 만들어 POST /ai/ingest/{signals,predictions,sentiment}
기본적으로 매매는 안 함(enable_auto_trade=False). 켜면 confidence ≥ 0.7 일 때만
BTC 를 자동매매(패턴 B).
   · 코드: bots/bot_a_ingest.py

─────────────────────────────────────────────────────────────────────────────
5-2. BotA_Trade — "AI 자동매매 봇" (실제 매수/매도, ENABLE_TRADE_BOTS=true 시 ON)
─────────────────────────────────────────────────────────────────────────────
역할 : model_trade.onnx(5분 후 예측)로 방향을 판단해 **실제로 사고파는** 핵심 봇.
주기 : TRADE_POLL_INTERVAL_SEC (기본 30초, WS 모드면 캔들마다)
매매 결정 알고리즘 (BTC 풀·ETH 풀 각각 독립 수행):
   1) 최신 피처로 model_trade 추론 → 코인별 예측 로그수익률 ret
   2) 게이트 통과 검사(둘 다 만족해야 거래):
         |ret| ≥ trade_min_log_return (0.0002 ≈ 0.02%)   ← 신호가 충분히 큰가
         confidence ≥ trade_min_confidence (0.55)          ← 충분히 확신하는가
   3) 방향: ret>0 → BUY(quote_to_base, USDT로 코인 매수)
            ret<0 → SELL(base_to_quote, 코인을 USDT로 매도)
   4) 수량(중요): 매도는 base 코인 수량(기본 0.02), 매수는 그 수량×현재가를 USDT로
      환산. (양쪽에 같은 숫자를 쓰면 한쪽이 풀 대비 과대해져 슬리피지 danger 로
      매번 스킵되는 문제를 방지)
   5) /swap/quote → slippage_level=danger 면 스킵, 아니면 /swap/execute
      (expected_revision 으로 옵티미스틱 락, 409 StaleQuote 면 재견적 후 재시도)
   6) 체결 후 풀별 쿨다운(trade_cooldown_sec, 5초) 동안 같은 풀은 추가 매매 대기
가드레일: 슬리피지 허용 200bps, danger 스킵, StaleQuote 재시도(3회), 풀별 쿨다운.
   · 코드: bots/bot_a_trade.py
   · 신뢰도 변환: _ret_to_confidence (5-2 의 confidence 와 동일 식)

─────────────────────────────────────────────────────────────────────────────
5-3. BotB_Noise — "시장 조성 봇" (AI 아님, ENABLE_TRADE_BOTS=true 시 ON)
─────────────────────────────────────────────────────────────────────────────
역할 : AI 예측 없이 **무작위로** 사고팔아 풀에 거래량·유동성을 공급. 덕분에 차트가
       살아 움직이고 슬리피지가 현실감 있게 형성됨(가상 시장 조성).
알고리즘:
   1) 3~8초 무작위 대기
   2) BTC/ETH 풀 중 하나를 무작위 선택, 매수/매도 50:50 무작위
   3) 수량 0.001~0.05 무작위 (매수는 현재가로 USDT 환산)
   4) 견적 → danger 면 스킵, 아니면 즉시 체결 (StaleQuote 면 재시도 없이 다음 사이클)
이 봇은 "옳은" 매매를 하지 않습니다. 시장에 잡음(noise)과 양방향 거래를 넣어
BotA_Trade 와 충돌시키는 상대역입니다.
   · 코드: bots/bot_b_noise.py

─────────────────────────────────────────────────────────────────────────────
5-4. BotC_LP — "유동성 공급 봇" (enable_lp_bot=true 시 ON)
─────────────────────────────────────────────────────────────────────────────
역할 : 매매가 아니라 유동성(LP)을 공급하는 봇. scope bot:lp.
알고리즘:
   1) 5분마다 풀 상태·24h ticker 조회
   2) 현재가와 24h 중간가의 괴리(spread)가 0.5% 초과면 불균형으로 판단
   3) /liquidity/quote-add → /liquidity/add 로 기준 금액(50) 유동성 공급
   · 코드: bots/bot_c_lp.py

─────────────────────────────────────────────────────────────────────────────
5-5. BotB_WS — "실시간(WS) 분석 봇" (use_websocket=true 시 ON)
─────────────────────────────────────────────────────────────────────────────
역할 : BotA_Ingest 와 같은 분석을 REST 폴링이 아니라 WebSocket 캔들 푸시 기반으로
       수행(저지연). 단일 WS 연결을 CandleEventBus 로 분배.
   · 코드: bots/bot_b_ws.py, services/event_bus.py

6. 매매 결정 파라미터 한눈에 보기 (config.py / .env)
================================================================================
  [시스템 A · 분석/선택적 자동거래]
    INGEST_INTERVAL_SEC          분석 주기 (기본 30~60초)
    enable_auto_trade            BotA_Ingest 자동거래 on/off (기본 off)
    trade_confidence_threshold   자동거래 최소 신뢰도 0.7

  [시스템 B · AI 자동매매 / 노이즈]   ※ ENABLE_TRADE_BOTS 로 일괄 on/off
    trade_min_log_return         0.0002   거래 트리거 최소 |예측수익률|
    trade_min_confidence         0.55     거래 트리거 최소 신뢰도
    trade_execute_amount_human   0.02     1회 거래 수량(base 코인 기준)
    trade_execute_slippage_bps   200      거래봇 슬리피지 허용(2%)
    trade_cooldown_sec           5        풀별 연속거래 최소 간격
    trade_poll_interval_sec      30       REST 폴링 주기
    noise_interval_min/max       3 / 8초  노이즈 봇 거래 간격
    noise_amount_min/max         0.001 / 0.05   노이즈 봇 거래량
    noise_slippage_bps           500      노이즈 봇 슬리피지 허용(5%)

  [모델/피처]
    seq_len_trade/scalper/swing/longterm   30 / 10 / 60 / 120
    candle_interval              5m  (학습 데이터와 반드시 동일)
    ema_alpha                    0.3 (출력 평활)

  · 부팅 시 거래봇은 BTC·ETH·USDT 를 모의 입금으로 자동 충전(main._ensure_trade_funds).
    거래하는 모든 풀의 base/quote 가 있어야 잔고부족 없이 양방향 매매 가능.


================================================================================
7. 학습 → 배포 → 재현 파이프라인 (train/)
================================================================================
"AI 가 과거 학습한 모델을 재현한다"는 취지를 실제로 구현하는 절차입니다.

  00_download_binance.py   Binance 3년치 BTC·ETH 5m 캔들 다운로드 (data/raw/*.csv)
  01_build_dataset.py      8피처·타깃 생성 + scaler 적합 (models/scaler*.pkl)
  02_train_model.py        GRU 4모델 학습 (Colab T4, 모델별 3~20분)
                           → 방향 정확도 BTC≥55%/ETH≥53% 목표
  03_export_onnx.py        학습된 모델을 ONNX 로 변환 (models/model_*.onnx)
  배포                     models/ 의 .onnx + scaler.pkl 을 엔진에 복사 → 봇이 재현

핵심: 학습은 과거 데이터로 "한 번" 끝내고, 봇은 그 결과를 "매번 재현(추론)"만 함.
그래서 같은 모델이 어떤 시장에서도 동일하게 행동하며, 사용자는 그 행동을 반복
관찰하며 "이 AI 는 이런 패턴에서 이렇게 판단하는구나" 를 학습할 수 있습니다.

  · 상세: train/traning_guide.md
